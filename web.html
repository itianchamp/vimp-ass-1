WELCOME TO DOCUMENTATION OF VCS IMPLEMENTATION USING GITHUB

OVERVIEW

This documentation includes how to create or set a new repository, how to push data in a repository, how to clone a repository.

>>CREATING/SETTING A NEW REPOSITORY

GIT INIT COMMAND:

After creating the repository on your github account we start initializing our repository by using git init command.

Here I use GIT INIT command to initialize the local directory as a git repository, GIT INIT command creates a new repository and also initializes the new and empty repository. This is the first command you will run in the new project.

GIT ADD COMMAND:

Next I used this command to add all the files to the folder. The git add command adds files to the staging area. Files in the staging area are those that you want to add to your next commit, changes are not actually recorded untill you run commit.

GIT STATUS COMMAND:

After using git status command i will now see all the files that i have stage for commit.It shows the state between the git add and git commit command. The git status command displays the state of repository and staging area, it allow us to see the tracked, untracked files and changes.

GIT COMMIT COMMAND::

After checking the status of files I use git commit command to commit the files that i have stage in my local repository. It is like a snapshot of repository. This is used for moving files from staging area to commit. It also contains a lot of meta data.

GIT REMOTE ADD ORIGIN COMMAND:

After commiting I add URL for the remote repository where your local repository will be pushed. To communicate outside the world GIT uses "remotes". There are repositories other than your local disk in which you can push your changes or pull from them. The command GIT REMOTE ADD ORIGIN creates a new remote called origin which is located at the URL you provided.

>>PUSHING DATA IN REMOTE REPOSITORY

GIT PUSH ORIGN MASTER:

Now i am pushing my code from my local repository to github repository by using GIT PUSH ORIGN MASTER command. This command push the commits from the local branch named master to the remote named orign. Once this is executed, all the stuff yhat you last synchronised with orign will be sent to the remote repository.

>>CLONING REPOSITORY

GIT CLONE COMMAND:

The git clone is a git command, which creates a clone/copy of an existing repository into a new directory. It is also used to create remote-tracking branches for each branch in the cloned repository. It is the most common command which allows users to obtain a development copy of an existing central repository.
